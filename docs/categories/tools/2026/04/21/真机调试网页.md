# 真机调试网页

## 背景

> 在PC端使用H5技术开发和调试时，由于主流浏览器Chrome、Safari、Firefox的devtools已经做的比较成熟，提供了较好的开发体验。但在移动端开发时，无法直接使用devtools。虽说H5技术具有跨平台性，理论上可以在PC上开发，直接运行于移动端，但事实是移动端webview内核的实现并非与PC端一模一样，有很多兼容性的问题只能在移动端重现，因此有一些场景不可避免的需要在移动端真机上进行开发和调试。那有没有办法让移动端H5开发和PC上有一样的体验呢？答案就是——Remote Debugging(远程调试)

## Remote Debugging

### 原理

- 本文重点介绍如何使用，关于devtools的工作原理，后面有时间会专门介绍。
- ![image-2026-04-21](/img/2026/04/21/1776738282589.png)

- Android 页面调试，优先使用 Chrome 自带的远程调试能力。
- iPhone 上的 Safari 页面调试，优先使用 Safari 的开发菜单。
- 如果希望继续用 Chrome 调试 iPhone 页面，可以通过代理工具把 iOS 的 WebKit 调试能力桥接到 Chrome DevTools。

## Android 真机调试

### 一、打开手机USB调试功能

1. 设置 -> 关于手机 -> 系统版本。
2. 连续点击系统版本，一般5次以内就会弹出“已进入开发者模式”的toast。
3. 设置 -> 系统 -> 开发人员选项 -> 打开USB调试
4. ![image-2026-04-21](/img/2026/04/21/1776738420311.png)

### 二、USB连接手机与电脑

1. 用数据线连接手机与电脑
2. 手机上弹出如下对话框，一定要选确定/允许。
3. ![image-2026-04-21](/img/2026/04/21/1776738465094.png)
4. 电脑上也会弹出授权对话框，同样要选允许。

### 三、开始调试

1. 手机chrome浏览器打开待调试页面。
2. 在PC chrome浏览器地址栏输入 <b>chrome://inspect/#devices</b> -> 找到待调试页面 -> 点击 <b>inspect</b> 打开devtools。
   ![image-2026-04-21](/img/2026/04/21/1776738580110.png)  
   ![image-2026-04-21](/img/2026/04/21/1776738601831.png)

3. 用chrome devtools调试
   ![image-2026-04-21](/img/2026/04/21/1776738612324.png)

## 用chrome devtools调试手机safari网页

原文给出的思路是先启动 WebKit 调试代理，再把调试端口接入 Chrome。

### 安装工具

```bash
brew install ios-webkit-debug-proxy
npm install -g remotedebug-ios-webkit-adapter
```

### 启动代理

```bash
ios_webkit_debug_proxy -f chrome-devtools://devtools/bundled/inspector.html
remotedebug_ios_webkit_adapter --port=9000
```

### Chrome 连接方式

1. iPhone 连接电脑后，先在 Safari 中打开目标页面。
2. 电脑端打开 `chrome://inspect`。
3. 在 `Configure` 中添加 `localhost:9000`。
4. 等待页面出现在列表中后，点击 `inspect` 即可。

## 适用场景

- 平时主要使用 Chrome DevTools，希望 iOS 也保持同一套调试习惯。
- 需要同时调试 Android 和 iPhone 页面，想统一入口。
- 想借助 Chrome 的元素面板、网络面板和控制台完成排查。

## 总结

- Android：Chrome 远程调试最直接。
- iOS Safari：Safari 开发菜单最稳定。
- iPhone + Chrome：通过代理桥接后也能调试。
- 真机调试前，先检查设备授权、调试开关和页面是否已打开，通常能省掉大部分排查时间。
